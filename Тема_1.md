### Создание нового локального репозитория с помощью Git Bash
### Результат
![img7](https://github.com/46acf5/labs/blob/main/img/%D0%9B%D0%B0%D0%B1_1/photo_2024-09-11_02-25-35.jpg)
## Вывод
Команда **git init**
  - Инициализирует новый репозиторий в текущей директории. Используется для начала работы с **git add**.
  - Добавляет файлы или их изменения в индекс для последующего коммита. Применяется, когда необходимо подготовить файлы для **git status**.
  - Показывает текущее состояние репозитория, включая изменения, готовые к коммиту. Используется для проверки перед **git commit**.
  - Сохраняет зафиксированные изменения в репозитории. Применяется для создания нового коммита с комментарием.
  - Отображает историю коммитов. Используется для анализа предыдущих изменений в проекте.

### Добавление изменений в стэш
### Результат
![img8](https://github.com/46acf5/labs/blob/main/img/%D0%9B%D0%B0%D0%B1_1/photo_4_2024-09-11_03-07-20.jpg)
## Вывод
**git stash** временно сохраняет текущие изменения в хранилище без коммита. Применяется, когда нужно переключиться на другую задачу без потери текущих изменений.
**git stash apply** применяет изменения из хранилища, не удаляя их из списка. Используется, когда нужно восстановить временно сохраненные данные.

## Вывод
- **git branch** позволяет создать новую ветку;
- **git checkout** переключает ветки или восстанавливает файлы из предыдущих коммитов;
- **git switch** позволяет переключиться на ветку;
- **git rebase -i HEAD~3** позволяет перебазировать ветку в конец целевой ветки. Интерактивно изменяет последние 3 коммита (опционально). Применяется для редактирования, объединения или изменения порядка последних коммитов;
- **git merge** создает новый коммит слияния, объединяя две истории, тогда как **git rebase** перезаписывает историю коммитов, что делает его более "чистым", но требует больше внимательности при разрешении.

## Вывод
Для того чтобы начать отслеживать (добавить под версионный контроль) новый файл, используется команда **git add**. Проверить успешность выполнения команды можно через **git status**.
Отправка изменений в чистый репозиторий делается при помощи команды **git remote add <repository_name> link**. Фиксация текущих изменений, в свою очередь, происходит при вызове команды **git push origin <branch>**.
**git push** отправляет изменения в удаленный репозиторий. **git pull** получает и сливает изменения из удаленного репозитория в локальный. Применяется для синхронизации с удаленным репозиторием.
